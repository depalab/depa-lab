# DEPA Research Lab — Official Website

React single-page app for the DEPA Lab at Morgan State University. Deployed to
GitHub Pages via the workflow in `.github/workflows/deploy.yml`.

## Run it

```bash
npm install
npm start          # local dev server
npm run build      # production bundle (what GitHub Actions runs)
```

## Project layout

```
.
├── .github/workflows/deploy.yml   GitHub Pages CI
├── package.json
├── tailwind.config.js
├── public/
│   ├── index.html                 HTML shell
│   └── images/                    All assets referenced by home.jsx
└── src/
    ├── index.js                   ReactDOM entry
    ├── index.css                  Tailwind directives + base
    ├── App.js                     Renders <DepaLabHomepage />
    ├── App.css                    Inter font, scrollbar styles
    └── components/
        └── home.jsx               The whole site
```

That's it. Everything the site needs lives here.

## Recent fixes

### Browser hang/crash

`home.jsx` listened for `mousemove` globally and called
`setMousePosition({x, y})` on every event. The state was read inline in JSX
(`radial-gradient(... at ${mousePosition.x}px ...)`), so each pixel of
movement re-rendered the entire ~2,500-line component, including 12 inline
research-detail pages, 12 team cards, and 9 project cards. Browsers froze.

Fix: removed the state, added an `overlayRef`, and the mousemove handler now
writes directly to `overlayRef.current.style.background` inside a
`requestAnimationFrame` callback. No React re-renders happen during mouse
movement, and updates are coalesced to one DOM write per frame.

### Per-render allocation cleanup

- `presentTeamMembers` and `pastTeamMembers` moved to module scope so they
  aren't re-allocated on every render.
- `ResearchComponents` (the 12-key map of inline detail pages) is wrapped in
  `useMemo([])` so its inner component functions aren't re-created on every
  render.
